# 介绍

这是一个我自己日常收集的一个集合，包括软件和书籍，未来还会加上一些电影，使用的库以及经常看待的公开课。

# 软件

### [Calibre](http://calibre-ebook.com/)

这是一个图书的管理工具，可以管理PDF、epud、mobi等格式的图书，并且还可以根据自己的需修改文档管理的结构。

### [VMware Workstation](http://www.vmware.com/products/workstation.html)

为什么不推荐VirtualBox呢？其实VirtualBox也是一个挺好的虚拟机平台，但是在图形界面下还是有点不如VMware，所有推荐这个软件作为虚拟机的工具。

### [Typora](http://www.typora.io/)

一个免费的编写Markdown的工具，功能十分的强大，相比Mac上有许多的Markdown的编写工具，在Windows上这样的工具就少了许多，以前都是使用Sublime加上插件来写的，但是自从发现这个软件之后就一直使用这个软件来进行编写Markdown了，并用其来编写日常的笔记。

### [SourceTree](https://www.sourcetreeapp.com/)

一个免费的图形化的Git客户端，当然也可以使用Github Desktop来进行管理，但是其显得比较笨重，而Git自带的Git GUI有显得比较鸡肋，所以我推荐这个作为日常的Git的管理工具。

### [Jetbrains](https://www.jetbrains.com/)

这是我推荐的一个编程工具全家桶，包括idea，webstrom还有就是pycharm都是十分好用的IDE。

### [VScode](https://code.visualstudio.com/)

现在有许多的编辑器，例如Sublime，Atom，Notepad++等，以上这几个我都用过，但是我发现VScode中个功能已经适合我的绝大部分时候的使用，最小的配置，最优的使用。当然在Linux上我还是推荐使用Vim或者是Emacs这两个工具，在Mac上推荐使用TextMate。

### [Sublime](https://www.sublimetext.com/)

进行前端开发的人应该特别喜欢这个文本编辑器，其丰富的插件，对于编写前端的页面代码十分有班长。很快快的开启速度，还是对于代码的查看很有帮助的，不过有了VScode后，我基本不是用Sublime了。

### [VIM](http://www.vim.org/)

一代上古神器，虽然现在大量使用IDE进行编程，但是在linux终端中还是得用到这个软件，真的算的上一个神器。如果加上各种插件，有些时候连一些所谓很牛的IDE也比不了。另一个神器是Emacs，主要是我不会用，不做太多评价。

### [印象笔记（Envernote）](https://www.yinxiang.com/)

对于笔记类的工具来说，有印象笔记、未知笔记，有道云笔记和OneNote，可谓是各有各的好处，但是对于我来说，我还是特别喜欢使用印象笔记进行记录我日常的学习的东西。但是由于印象笔记技术的限制对于Markdown的支持不是特别好（虽然有马克飞象），所以在去年开始，我开始使用Github来管理我的笔记了。

### [DISM++](https://www.chuyu.me/zh-Hans/index.html)

一个Windows上的管理软件和清洁软件，相当不错的一个东西。同类型的清洁软件还有CCleaner，Wise Clean，这些都是不错的东西。

### [SmartGit](http://www.syntevo.com/smartgit/)

最近发现的一个管理Git相当好的客户端，当然不付费的情况下只能在非商业项目上进行使用，其中对代码的review功能做的比SourceTree好一些，相比下来，最近一段时间我比较喜欢使用SmartGit管理我自己的项目。



# 书籍

## 技术书籍

### Java编程思想

虽然现在大部分的时候用的是JDK8进行编程，但是这本书中对于JDK的一些理解是相当不错的，建议有一定的基础知识后再进行阅读。

### Java核心技术卷

许多人都推荐过各种各样的Java入门书籍，包括视频。但是个人认为，目前的Java入门书籍还是《Java核心技术卷》比较好，第九版是基于JDK7的，第十版是基于JDK8的，都写的特别好，不过比较厚，看的时候可能有一种畏难情绪。

### Spring实战

关于Spring的中文书籍，我个人是看Spring实战第三版入的这个门，这个是基于Spring 3.0.5的，讲述了Spring中的绝大部分用法，并且简洁简练，但是没有讲述核心原理。第四版正在看中，暂时不做评价。

### Spring入门经典

一本入门Spring比较详细的书籍，基本讲述了绝大部分Spring的核心理念及其用法，并且有相当完备的例子。在中文书籍中，这算是一本比较好的入门书籍了。也是没有讲述核心原理，入门不错。

### 架构探险

本书描述了一个Smart Framework框架的设计与实现，从中可以学习到许多的各类框架设计和实现的原理，原理是一致的，但是实现是多种多样的。这本书有一个缺点就是大量的代码，然后讲解的少，需要自己一点一点的去思考设计的思想和原理，还是有点麻烦的。

### C程序设计语言

这本是丹尼斯·里奇 和布莱恩·柯林汉的书籍写的书，虽然过了许多年，但是还是C语言入门比较好的书籍，后面对于早期C语言标准库的描述还是比较好的解释的

### 七周七语言

看到七周七语言，还以为是一本包教包会的书，但是不是，这本书用七门语言，让你体验了多种编程语言和编程泛型。目前刚出第四版。

### Java8函数式编程

一本动物园的书，如果说这本书很好呢，难。但是作为一个队Java8的函数式编程的了解已经足够用了，更多的函数式编程的思想还是得看《计算机程序的构造和解释》。

### 算法（第四版）

如果说贴近实际使用的算法书是哪本，我肯定会说是现在我看了一遍半的《算法》了，一个具体的实现，一个具体的讲解，外加在Coursea上普林斯顿的公开，已经足够了解计算机常见算法的含义及使用了，包括如何去实现。

### 数据结构与算法分析

这套书共有三个版本，分别是C的，C++的和Java的，这应该是最贴近大学上课时候算法学习需要的书了，其中的一些想法，与国内的教程相比，都是比较好的。授人以鱼不如授人以渔就是说的这种类型的书了，讲的比较好懂，我看C讲解AVL树的时候没看懂，但是换到Java版的实现的时候我就清楚了。

### 算法导论

这是我买的第一本算法书，也是目前我没看完的一本算法书，不是这本书不好，而是我看到里面的数学推理就头疼，看了200多页后放弃了。但是后来我喜欢上数学那是另外一本书给我的帮助和信心，所有在接下来的日子里，有时间我就会再把这本书看完的。

### MySQL必知必会

学习MySQL的基础的一本比较好的书籍，详细的介绍了MySQL中的SQL语句和编程。做为基础的学习MySQL的操作和使用已经基本够了，基本涵盖的MySQL的所有语法和知识点了。

### Java Persistence with MyBatis3

这是一本介绍MyBatis如何使用的小手册，主要我看中的是其有一个完整的例子，如果进行学习的话，官方的文档已经是相当好的一个学习参考了，剩下的就是自己如何去实践，然后在实践中进行总结了。

### Python编程从入门到实战

这本书前半部分讲述了Python的基本语法和使用，而后半部分则是通过一个个工程示例讲述了如何编写一个Python的应用程序。这本书的主要价值我认为是在后面的编程实战部分，只有通过大量的编程练手，才能熟能生巧。

### 全端web开发

怎么说这本书的，这本书写的想的不错，对基于Java后端的全栈做了一个很详细的简介。作为我的一个学习方面的指引是想的不错的，只是看到这本书的时间有点晚，其中有的东西还没有了解过，只能从现在开始慢慢的进行接触了。

### C++ Primer

有轮子哥的保佑，这本书应该是入门学习C++比较好的书了，虽然这本书是一本大部头，但是如果真心想要学习C++的话，这是一本十分好的书籍了，要是想考试过关的话就没有那个必要了，找一本相关的书来看就行了。剩下的《C++编程思想》就看自己的需要再去看了，如果想要深入一下还是有必要去看看这一本书的。

### 计算机科学的基础

这是一本免费的书籍，如果觉得自己大学的时候没学好，在工作中发现当年那些自己认为没有用的东西还是有用的话，可以考虑重新读一遍这本书。虽然600多页的一本书，但是几乎涵盖了计算机方面的基础知识，感觉还是不错的。

### 计算机科学导论

任何学科都是需要有导论的，计算机方面也是一样的。看完这本书不需要多长的时间，但是可以让你对计算机和计算机学科有一定的了解，以免在以后的过程中会有不必要的尴尬发生。所以导论方面的书还是有必要看一看的。

## 科技其他

### 微积分之倚天剑

### 微积分之屠龙宝刀

如果说对数学没兴趣的话，看了吴军老师的书还是对数学没有多大的兴趣，可以看看这两本书，看完之后，不仅可以让你喜欢上数学，还会让你喜欢上用数学的思维去看待东西，当然考试也是逢考必过的那种了，O(∩_∩)O哈哈~。当然我隔几天准备看看《数学女孩》，到时候再说那本书更好了

### 世界是数字的

看名字就知道这是一本科普类型的书籍，文章从硬件、软件和通信三大块讲述了计算机中的基础知识，对于计算机的了解，还是比较有意思的一本书籍。

### 数学之美

吴军老师的一本书，这本书更像是一本科普类型的书籍，很久之前读的了。介绍了数学在整个计算机中的应用，因为吴军老师是做搜索的，其介绍的大部分数学知识还是和搜索有关系。

### 浪潮之巅

这本书也是吴军老师写的，写在中国所谓的移动互联网出现之前，其中讲述了许多在计算机界十分闻名的公司是如何诞生的，也有作者对于一些公司消亡的思考，值得一看。一句话就是，泡沫的破碎不是件坏事，其代表着另外一个时代的来领。

### 编程之魂

采访了25位语言的发明人，你从中可以知道为什么要发明这些语言，以及作者在发明这些语言的过程中的各种想法，以及他们对编程的看法，都是首屈一指的黑客，为什么不了解一下他们的想法呢？

###  编码：隐匿在计算机硬件背后的语言

了解计算机，不如从底层开始了解计算机，接触核心，这才有乐趣。这本书用形象的语言描述了计算机的组成，拿手电筒来解释计算机中的门，这个想法不错吧。

### 黑客与画家

这本书和传说中的一样，是本好书，但是，我没看懂。这本书是作者博客和一些演讲的集合，中间穿插了一些他公司创建的始末，也讲了一些称为一个优秀开发者所具备的素质和需要注意的地方。总之，有一点很对，程序员和画家一样，做的都是创造性的工作，画家有好有坏，程序员也是一样的。

### 技术之瞳

这是一本阿里的面试书籍，我推荐这本书的原因不是因为阿里的原因，而是其中给了一个学习的路线和方向，指明了计算机相关学生在大学毕业的时候应该知道的理论方法的知识。当然的，这本书上还是有许多的保留和缺陷的，一个模子，适合借鉴，不适合完全照搬。

## 小说

### 豆瓣：流行的秘密

web 2.0时代的开端，信息孤岛的开始，看待现在的豆瓣，貌似可以看见其中知识。这是我读的第一本对于互联网的公司的书籍，一个信息孤岛时代的开始，但是我认为豆瓣也有可能成为信息孤岛的墓地。

### 从0到1

概念来自与博弈论中国的零和博弈，当然书是PayPal的创始人写的。如果说是创业圣经有点过头了，因为创业的成功率不到10%，能看到一些失败的原因。

### 失乐园

这是渡边淳一的一本关于爱情的书籍，故事的一个背景是真实在日本发生过的。其他的，看了书就知道了，不多说了

### 相约星期二

这是我前几年读的一本书，其中的内容，对生活，对于面对死亡时候的人都心里，都描述的特别的好。一个做记者的学生对其老教授“上课”的访谈，其中的一些内容的描述，对于自己对于我现在这种焦虑的生活，以及对于毕业后就有可能失业的人的心情还是有教的帮助的。读了许多遍，每次看一遍，都有不同的看法，生活还是那样吧！！！

### 通往内心心灵安静之路

这是我高中时候买的一本书，当时应该是处在青春期的叛逆时候吧，觉得读书没有任何的意义，然后自己的心中还充满了各种各样的焦虑心情。当时喜欢看书，虽然在当时被看做是一个笑话，心中的焦虑心情也是十分大的。现在也是越实习越越感到焦虑，然后放假回来的时候专门把这本书带了回来准备在看一遍。

### 安妮宝贝

说是本书呢，也算是，但是也是一个作者的名字。她写的书算是陪伴了我的初中和高中吧。我先走从事计算机的工作也和这个事情有一些分不开的关系吧。安妮宝贝的10年前的书我是十分喜欢的，虽然那些文章一直被许多人批评，但是我个人还是十分喜欢这些散文和小说的。

## 传记

### 乔布斯传

一个iPhone 4的惊艳设计，一个自传，让我认为我以后的生活应该是编程，实现一个伟大的产品。当然一句话：是买一辈子的可乐还是来造就一个伟大的公司。

### 别闹了，费曼先生

## 公开课

### CS50

这是哈佛大学的一门计算机的入门课程，在网易上有2012年的视频的翻译，当时我还是建议去edx上面去看最新的，课程开始的时间是每年的九月份，自主学习。我基本每年都会看一遍，虽然知识点都是重复的，但是有些东西用不同的方法来讲的确挺有意思的。

### 算法

这个是普林斯顿大学推出的公开课，总共是有两个部分组成，其配套的书是《算法》（第四版），并且加上课本官网上面的资料，对于进行算法学习来说是比较好的一门课程，如果只是想快速的了解算法的东西，看视频就够了（我想，程序员应该把算法搞好吧）。

### CS 6.001

这个是麻省理工学院的计算机系的入门课程，名称是Introduction to Computer Science and Programming Using Python，这个课程以前的配套课本就是封面是lambda符号的那本，《计算机程序的构造和解释》，貌似我记得从07年开始就换成Python了。作为计算机的入门课程，我认为在配上基本计算机导论方面的书籍，就是一个完美的选择了。
